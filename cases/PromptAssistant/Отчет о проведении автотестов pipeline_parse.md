## 4. Охваченные сценарии
- Позитивные:
  - корректный JSON (одиночный объект, массивы промптов);
  - корректный разбор выходов шагов 2, 6, 7;
  - разделение промптов по меткам `[Промпт N]`;
  - детекция email-артефактов (плоская и вложенная формы).
- Негативные:
  - битый JSON и ситуации, когда объект извлечь невозможно;
  - пустой/невалидный вход;
  - неожиданные типы данных.
- Граничные:
  - escape-последовательности и вложенные скобки в строках;
  - fallback-логика при неудачном парсинге;
  - нормализация числовых значений и преобразования в `int`.
