## 1. Цель документа
Зафиксировать полный перечень сценариев и тестовых данных для масштабного тестирования Competitor AI в соответствии с `docs/План автотестов.md`.

## 2. Область применения
Документ покрывает сценарии для:
- backend endpoint-ов;
- сервисного слоя;
- frontend-клиентской логики;
- desktop-клиента;
- регрессионного и релизного контура.

## 3. Структура тестовых данных
Основные smoke-данные (текущие) остаются в:
- `tests/data/texts`
- `tests/data/images`
- `tests/data/pdfs`
- `tests/data/urls.txt`

Полномасштабный контур (новый) расположен в:
- `tests/data/fullscale/texts`
- `tests/data/fullscale/images`
- `tests/data/fullscale/pdfs`
- `tests/data/fullscale/urls`
- `tests/data/fullscale/history`

Общий запуск выполняется через:
- `python tests/run_quality_checks.py`

Порядок этапов:
1. `smoke_check.py` (базовая стабильность API),
2. `fullscale_check.py` (расширенные аспекты и негативные сценарии).

## 5. Наборы данных по сценариям

### 5.1 Тексты
- `tests/data/fullscale/texts/positive/landing_long_01.txt`
- `tests/data/fullscale/texts/positive/landing_long_02.txt`
- `tests/data/fullscale/texts/positive/mixed_offer_03.txt`
- `tests/data/fullscale/texts/negative/text_too_short.txt`

### 5.2 URL
- `tests/data/fullscale/urls/positive_urls.txt`
- `tests/data/fullscale/urls/negative_urls.txt`

### 5.3 PDF
- Валидные PDF: использовать набор `tests/data/pdfs/*.pdf`.
- Невалидные PDF:
  - `tests/data/fullscale/pdfs/negative/invalid_pdf.pdf`
  - `tests/data/fullscale/pdfs/negative/not_a_pdf_with_pdf_extension.pdf`

### 5.4 Изображения
- Валидные изображения: использовать набор `tests/data/images/*.png`.
- Невалидные изображения:
  - `tests/data/fullscale/images/negative/not_image.jpg`
  - `tests/data/fullscale/images/negative/unsupported_image.svg`

### 5.5 История
- `tests/data/fullscale/history/history_seed.json` — стартовая заготовка для тестов миграции/очистки.

## 6. Критерии прохождения
- Позитивные сценарии возвращают успешный статус и валидную структуру ответа.
- Негативные сценарии возвращают предсказуемую ошибку в корректном формате.
- После `DELETE /history` очищаются:
  - `data/history.json`;
  - `data/uploads/*`;
  - `data/screenshots/*`;
  - `data/tmp_uploaded.pdf`.
- Результаты фиксируются в `docs/Отчет о тестировании.md`.
