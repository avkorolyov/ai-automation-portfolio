# Руководство по установке и запуску

> Документ содержит технические инструкции для локального запуска, проверки и сборки кейса `Competitor AI`.

## 1. Предварительные требования
- Python 3.11+ (рекомендуется 3.13.x).
- Node.js 18+ и `npm` (для frontend-тестов и coverage).
- Установленный браузер для Selenium-контуров e2e.
- Доступ в интернет для внешних URL-сценариев и AI-провайдера.

## 2. Установка окружения

### 2.1 macOS / Linux
```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

### 2.2 Windows (PowerShell)
```powershell
python -m venv venv
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
Copy-Item .env.example .env
```

### 2.3 Переменные окружения
Заполни `.env`:
- `POLZA_BASE_URL=https://polza.ai/api/v1`
- `POLZA_AI_API_KEY=...`
- опционально `APP_PORT=8010`, если `8000` занят.

## 3. Запуск сервиса

### 3.1 Backend
```bash
source venv/bin/activate
python backend/run.py
```

После запуска доступны:
- UI: `http://127.0.0.1:<APP_PORT>/`
- Swagger: `http://127.0.0.1:<APP_PORT>/docs`
- Health-check: `http://127.0.0.1:${APP_PORT:-8000}/health`

Проверка доступности:
```bash
curl http://127.0.0.1:${APP_PORT:-8000}/health
```

### 3.2 Desktop-клиент
Установи desktop-зависимости:
```bash
source venv/bin/activate
pip install -r desktop/requirements.txt
```

Запуск:
```bash
source venv/bin/activate
python desktop/main.py
```

## 4. Примеры API-запросов

### 4.1 Анализ текста
```bash
curl -X POST http://127.0.0.1:${APP_PORT:-8000}/analyze/text \
  -H "Content-Type: application/json" \
  -d '{"competitor_name":"BrandLens AI","text":"...длинное описание конкурента..."}'
```

### 4.2 Парсинг сайта
```bash
curl -X POST http://127.0.0.1:${APP_PORT:-8000}/parse/demo \
  -H "Content-Type: application/json" \
  -d '{"url":"https://example.com"}'
```

### 4.3 Анализ изображения
```bash
curl -X POST http://127.0.0.1:${APP_PORT:-8000}/analyze/image \
  -F "competitor_name=universal" \
  -F "file=@./tests/data/images/sample.jpg"
```

### 4.4 Анализ PDF
```bash
curl -X POST http://127.0.0.1:${APP_PORT:-8000}/analyze/pdf \
  -F "competitor_name=universal" \
  -F "file=@./tests/data/pdfs/sample.pdf"
```

## 5. Тестирование и качество

### 5.1 Единый тестовый пайплайн
```bash
source venv/bin/activate
python tests/run_quality_checks.py
```

Состав:
- `tests/smoke_check.py`
- `tests/fullscale_check.py`
- `tests/contract_api_check.py`
- `tests/resilience_check.py`
- `tests/security_check.py`
- `tests/e2e_ui_check.py`

### 5.2 Формальный coverage-gate
```bash
source venv/bin/activate
python tests/run_coverage_checks.py
```

Артефакты:
- `tests/artifacts/coverage/coverage.xml`
- `tests/artifacts/coverage/frontend/coverage-summary.json`
- `tests/artifacts/coverage/.coverage*`

## 6. Сборка desktop-релиза
```bash
# macOS
python desktop/build.py --profile macos --clean

# Windows
python desktop/build.py --profile windows --clean
```

Результат:
- `dist/Competitor AI.app` (macOS)
- `dist/Competitor AI.exe` (Windows)
- `release/Competitor AI-<profile>-manifest.json`

Важно:
- кросс-компиляция в PyInstaller не поддерживается;
- сборку выполнять на целевой ОС.
