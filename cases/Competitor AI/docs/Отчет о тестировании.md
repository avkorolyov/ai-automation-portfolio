## 1. Цель документа
Зафиксировать рабочий формат отчета по тестированию и минимально достаточный набор автосценариев, позволяющий поддерживать покрытие автотестами кода проекта выше 80%.

## 2. Область применения
Документ используется QA и командой разработки при подготовке релизов и регрессионных прогонов.  
Отчет оформляется для каждой релизной итерации и хранит фактические результаты, дефекты и решение по выпуску.

## 3. Общая информация
- Релиз / версия: pre-release (локальный прогон перед полномасштабным тестированием)
- Период тестирования: 2026-04-22
- Тестовое окружение: локальный запуск на macOS (backend + tests)
- Ответственный QA: AI-assisted прогон (техническая верификация)
- Версия Python: 3.13.2
- Commit / ветка: недоступно (проект запущен вне git-репозитория)

## 4. Целевые критерии качества
- Обязательный критерий: суммарное покрытие строк кода проекта (backend + frontend) **> 80%**.
- Минимальные пороги по контурам:
  - backend line coverage (`coverage.py`): `>=80%`;
  - frontend line coverage (`vitest v8`): `>=80%`.
- Для критических пользовательских потоков (text/image/pdf/parse/history) — покрытие ветвей позитив/негатив.
- Все критические сценарии из раздела 6 должны иметь статус `PASS`.
- При наличии `BLOCKER` или `CRITICAL` дефекта релиз не рекомендуется до исправления и ретеста.

## 5. Объем тестирования
- Backend API:
  - `POST /analyze/text`
  - `POST /analyze/image`
  - `POST /analyze/pdf`
  - `POST /parse/demo`
  - `GET /history`
  - `DELETE /history`
- Web UI (`frontend/app.js`, `frontend/styles.css`, `frontend/index.html`): клиентская валидация и корректная обработка API-ошибок.
- Desktop (`desktop/main.py`, `desktop/api_client.py`, `desktop/build.py`): доступность backend, маршрутизация сценариев, обработка ошибок клиента.
- Smoke-контур (`tests/smoke_check.py` + `tests/data/*`).
